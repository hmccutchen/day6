Working with Git

1. add files to your project

This can be done via the touch command or though the interface as usual

Example: touch index.html && touch main.css

2. Add new or modified files to staging

Once we add new files or modify existing files we can tell git to track them by adding them to staging for the next commit (thinkof cmmits as checkpoints) This will let git know what files the next checkpoint will consit of.

git add < filename >

OR

git add .

Your files are now on stage to be included in the next commit/checkpoint

3. Add a commit and a commit message

with your changes staged, you can create the checkpoint in your with the following command

git commit -m "commit message"

This command creates the checkpoint for git! pretty simple

4. push your code

Finally push your changes up to Github with the following command:

git push

If you get an error you may have not initialized your git repo w/ git init
or you may have initialized, but you have not yet modified any files since then using:

git add < filename >

and

git commit -m "commit message"
